This program shows the use of the \char`\"{}radio\char`\"{} bsp module. 

Since the bsp modules for different platforms have the same declaration, you can use this project with any platform.

This application places the mote in receive mode, and prints, over the serial port, all information about the received packet. The frame printed over the serial port for each received packet is formatted as follows\+:
\begin{DoxyItemize}
\item \mbox{[}1B\mbox{]} the length of the packet, an unsigned integer
\item \mbox{[}1B\mbox{]} the first byte of the packet, an unsigned integer
\item \mbox{[}1B\mbox{]} the receive signal strength of tehe packet, an signed integer
\item \mbox{[}1B\mbox{]} the link quality indicator, an unsigned integer
\item \mbox{[}1B\mbox{]} whether the receive packet passed C\+RC (1) or not (0)
\item \mbox{[}3B\mbox{]} closing flags, each of value 0xff
\end{DoxyItemize}

You can run the \hyperlink{01bsp__radio__rx_8py}{01bsp\+\_\+radio\+\_\+rx.\+py} script to listen to your mote and parse those serial frames. The application can connect directly to the mote\textquotesingle{}s serial port, or to its T\+CP port when running on the Io\+T-\/\+L\+AB platform.
